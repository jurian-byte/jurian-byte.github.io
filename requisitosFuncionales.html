Reglas de negocio

Requerimientos funcionales

RF1. El sistema maneja un registro de clientes el cual almacenará el nombre del cliente, usuario, número telefónico y contraseña.

RF2. El sistema maneja un registro de productos el cual almacenará el nombre del producto, precio, descripción, id del proveedor, nombre de la imagen y el id de categoría al que pertenece.

RF3. El sistema maneja un registro de proveedores el cual almacenará el nombre del proveedor y teléfono.

RF4. El sistema maneja un registro de ventas el cual almacenará el usuario del cliente, el id del producto, el nombre del producto, la descripción del producto, el precio del producto, la categoría del producto, la imagen del producto, el precio calculado, la cantidad del producto, el total pagado, la fecha de la venta, el tipo de transacción y la forma de pago.

RF5. El sistema maneja un registro de productos añadidos al carrito en el cual se almacenará el usuario del cliente, el id del producto, el nombre del producto, la descripción del producto, el precio del producto, la categoría del producto, la imagen del producto, el precio calculado, la cantidad del producto y la disponibilidad.

RF6. A cada registro que maneja el sistema se le asignará un id único, que será utilizado para identificarlo en todos los procesos que se realicen sobre este.

RF7. No se permitirá el registro de algún dato con campos incompletos, estos deben ser obligatorios.

RF9. El sistema controlará el acceso y, según el usuario y la contraseña, identificará si el usuario es administrador o cliente.

RF10. Solo el administrador podrá añadir información, visualizarla, modificarla y eliminarla, para cualquiera de los registros, menos de carrito.

RF11. El sistema debe poder emitir un archivo con extensión PDF con información de los productos cuando el administrador elige la opción de generar PDF.

RF12. El sistema debe poder leer, modificar, agregar y eliminar datos en archivos xml.

RF13. Cuando un usuario del tipo cliente elige la opción de logout, el sistema eliminará los registros almacenados del carrito.

RF14. El sistema debe validar los campos de los formularios, para campos numéricos como cantidad solo debe permitir introducir números.

RF15. El sistema permite al cliente elegir la cantidad de un mismo producto, al agregarla al carrito mostrará una pantalla con los datos del producto que se eligió para confirmar que se agregue al carrito.

RF16. El sistema permitirá visualizar los productos agregados al carrito y poder eliminarlos de él, antes de generar la venta.

RF17. El sistema cuenta con un buscador de productos que permite buscar directamente productos. A partir de la búsqueda se mostrará un listado de productos que cumplan los criterios indicados.

RF18. El sistema cuenta con buscadores por categoría. A partir de la categoría seleccionada se mostrará un listado de productos que cumplan con el criterio seleccionado.

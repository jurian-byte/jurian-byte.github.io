Casos de uso

Identificación CU1

Nombre: Identificación en la aplicación.

Identificador: CU1

Descripción: Identificarse en la aplicación para poder acceder a ella

Actores: Usuario y sistema

Precondiciones: Tener usuario, acceso y permisos a la plataforma

Post-condiciones: El usuario entra a la plataforma y puede interactuar con los elementos

Disparador: Selección de opción de login en el sistema

Curso básico de acción:
1. El sistema muestra un formulario con el nombre de usuario, contraseña y número telefónico
2. El usuario introduce los valores para los campos
3. El sistema valida que el identificador de cliente generado es correcto y es diferente de otro [Curso alt A]
4.Se presenta la pantalla principal del sistema con los elementos con los que puede interactuar el usuario

Curso alternativo A:
El sistema determina que el registro no es correcto A.3 El sistema arroja un mensaje indicando que el registro es incorrecto
A.4 El usuario escoge la opción de regresar al login
A.5 El caso de uso continúa en el paso 1 en el curso básico de acción

Busqueda articulos CU2

Nombre: Búsqueda de artículos en catálogo

Identificador: CU2

Descripción: Buscar un artículo específico mediante la opción de buscar en catálogo

Actores: Usuario y sistema

Precondiciones: Estar dentro de la aplicación de tienda online

Post-condiciones: El cliente encuentra una lista con los artículos que cumplen la búsqueda realizada

Disparador: Selección de opción de buscar artículos

Curso básico de acción:
1. Dentro de la aplicación, el cliente elige la opción para buscar artículos en el sistema
2. El sistema muestra un formulario para introducir las palabras de búsqueda
3. El cliente introduce los valores
4. El sistema busca los artículos que cumplen con el criterio indicado [Curso alt A]
5. El sistema muestra los artículos en pantalla

Curso alternativo A: El sistema no encuentra alguna coincidencia
A.4 El sistema muestra en pantalla un mensaje diciendo que ningún elemento coincide
A.5 El caso de uso termina

Consultar catalogo CU3

Nombre: Consultar catálogo

Identificador: CU3

Descripción: El cliente navega por la lista de elementos del catálogo disponible

Actores: Usuario y sistema

Precondiciones: Estar dentro de la aplicación de tienda online

Post-condiciones: El usuario navega por la jerarquía de catálogo y su lista de artículos

Añadir articulos CU4

Nombre: Añadir artículos a solicitud

Identificador: CU4

Descripción: El usuario agrega a la solicitud los artículos que le interesan

Actores: Usuario y sistema

Precondiciones: Haber realizado una búsqueda de artículos en el catálogo, bien por navegación o bien por búsqueda directa y estar registrado en el sistema

Post-condiciones: El artículo seleccionado se añade a una nueva solicitud

Disparador: El cliente selecciona la opción de añadir un artículo a una solicitud

Curso básico de acción:
1. El cliente selecciona el/los artículos en que está interesado
2. El cliente pulsa en añadir el artículo a una solicitud [Curso alt A]
3. El sistema crea una nueva línea en la solicitud
4. El sistema indica al usuario que se ha procedido a añadir el artículo a su solicitud de compra

Curso alternativo A: El artículo ya existía en la solicitud
A.2 El sistema suma las cantidades anteriores a las actualmente añadidas y no se crea una nueva línea en la solicitud
A.3 El caso de uso continúa en el paso 4 en el curso básico de acción

Enviar pedido CU5

Nombre: Enviar pedido

Identificador: CU5

Descripción: El cliente envía su pedido de artículos deseados

Actores: Usuario y sistema

Precondiciones: El cliente debe haberse identificado en el sistema y deben existir artículos en su solicitud

Post-condiciones: El pedido es creado en el sistema

Disparador: El cliente selecciona la opción de enviar pedido

Curso básico de acción:
1. El cliente selecciona enviar la solicitud actual
2. El sistema crea un pedido a partir de los datos de la solicitud
3. El sistema indica al cliente que se ha procedido a crear el pedido correctamente

Alta de cliente CU6

Nombre: Alta de cliente

Identificador: CU6

Descripción: El cliente desea darse de alta en el sistema

Actores: Usuario y sistema

Precondiciones: El cliente debe encontrarse navegando dentro de la aplicación

Post-condiciones: El nuevo cliente es creado en el sistema

Disparador: El cliente selecciona la opción de darse de alta en la plataforma

Curso básico de acción:
1. El cliente selecciona la opción de darse de alta en el sistema
2. El sistema le muestra un formulario con los datos que debe rellenar para poder acceder al sistema
3. El cliente rellena los datos del formulario y pulsa el botón de aceptar [Curso alt A]
4. El sistema comprueba que todos los datos son correctos [Curso alt B]
5. El sistema indica al usuario que el proceso de alta ha sido correcto y ya puede acceder a la plataforma con sus datos

Curso alternativo A: El cliente no rellena todos los datos solicitados
A.3 El sistema muestra un mensaje de error al cliente indicando que hacen falta datos
A.4 El caso de uso continúa en el paso 2 en el curso básico de acción

Curso alternativo B: El sistema determina que los datos son incorrectos o ya existen
B.4 El sistema muestra un mensaje de error al cliente indicando que los datos son incorrectos o ya existen
B.5 El caso de uso continúa en el paso 2 en el curso básico de acción

A.3 El sistema muestra un botón para salir de la opción de contacto
A.4 El caso de uso termina

Administración de registros del catalogo CU8

Nombre: Administración de registros del catálogo

Identificador: CU8

Descripción: Control de registros en el catálogo del sistema

Actores: Administrador y sistema

Precondiciones: Haber realizado una identificación válida en el sistema como administrador

Post-condiciones: El administrador da de alta, baja ó modifica registros en el catálogo de la tienda

Disparador: Selección de opción de administrar catálogo

Curso básico de acción:
1. Dentro de la aplicación, el administrador elige la opción de administrar catálogo
2. El sistema muestra un CRUD de los artículos que existen en el catálogo y muestra tres opciones que se pueden elegir: alta de producto, baja de producto o modificación de producto.
3. El administrador pulsa en las opciones disponibles por cada producto
4. El sistema muestra un mensaje para volver a la pantalla principal
5. El administrador elige la opción de regresar

Curso alternativo A: El administrador elige la opción de dar de alta producto

A.3 El sistema muestra un formulario con los datos requeridos para dar de alta un producto

A.4 El administrador rellena los campos requeridos y envía los datos

A.5 El sistema muestra en pantalla que se ha agregado exitosamente el artículo

A.6 El caso de uso continúa en el paso 4 en el curso básico de acción

Curso alternativo B: El administrador elige la opción de dar de baja un producto
A.3 El sistema muestra en pantalla que se ha eliminado exitosamente el artículo elegido
A.4 El caso de uso continúa en el paso 4 en el curso básico de acción

Curso alternativo C:El administrador elige la opción de modificar un producto
A.3 El sistema muestra un formulario con los datos actuales del artículo elegido
A.4 El administrador modifica los datos que desee y pulsa en enviar
A.5 El sistema muestra en pantalla que se ha modificado el registro con éxito
A.6 El caso de uso continúa en el paso 4 en el curso básico de acción

Generar clave del producto CU11

Nombre: Generar clave de producto

Identificador: CU11

Descripción: Generación de claves únicas para productos

Actores: Sistema

Precondiciones: Se debe enviar una petición de dar de alta producto

Post-condiciones: Se crea una clave única para el producto

Disparador: El administrador envía una solicitud para dar de alta un producto

Curso básico de acción:
1. El sistema crea una cadena vacía para almacenar datos de la clave
2. El sistema obtiene el número de compras en el mes en que se haga el alta, le suma un valor de 1 y finalmente lo agrega al comienzo de la cadena [Curso alt A]
3. El sistema agrega a la cadena la letra (mayúscula) del mes actual en que se realiza el alta. Teniendo en cuenta que la letra correspondiente sigue el siguiente criterio: enero = A, febrero= B y así consecutivamente hasta diciembre.
4. El sistema agrega a la cadena el número que corresponde al año en que se realiza el alta. Siguiendo el siguiente formato: 9 para el año 2019, 20 para el año 2020 y 21 para el año 2021.
5. El sistema obtiene cuantos productos similares se han vendido en el mes y año actual, le suma el valor de 1 y lo agrega a la cadena [Curso alt B]
6. Se crea exitosamente la clave del producto

Curso alternativo A: No se ha realizado una compra en el mes actual
A.3 El sistema agrega el valor 1 al comienzo de la cadena vacía
A.4 El caso de uso continúa en el paso 3 en el curso básico de acción

Curso alternativo A: No se ha realizado una compra de un producto similar en el mes y año actual B.3 El sistema agrega el carácter 1 a la cadena B.4 El caso de uso continúa en el paso 6 en el curso básico de acción

Casos de uso

CU1

Nombre: Identificación en la aplicación.

Identificador: CU1

Descripción: Identificarse en la aplicación para poder acceder a ella

Actores: Usuario y sistema

Precondiciones: Tener usuario, acceso y permisos a la plataforma

Post-condiciones: El usuario entra a la plataforma y puede interactuar con los elementos

Disparador: Selección de opción de login en el sistema

Curso básico de acción:
1. El sistema muestra un formulario con el nombre de usuario, contraseña y número telefónico
2. El usuario introduce los valores para los campos
3. El sistema valida que el identificador de cliente generado es correcto y es diferente de otro [Curso alt A]
4.Se presenta la pantalla principal del sistema con los elementos con los que puede interactuar el usuario

Curso alternativo A:
El sistema determina que el registro no es correcto A.3 El sistema arroja un mensaje indicando que el registro es incorrecto
A.4 El usuario escoge la opción de regresar al login
A.5 El caso de uso continúa en el paso 1 en el curso básico de acción

CU2

Búsqueda de artículos en catálogo

Identificador: CU2

Descripción: Buscar un artículo específico mediante la opción de buscar en catálogo

Actores: Usuario y sistema

Precondiciones: Estar dentro de la aplicación de tienda online

Post-condiciones: El cliente encuentra una lista con los artículos que cumplen la búsqueda realizada

Disparador: Selección de opción de buscar artículos

Curso básico de acción:
1. Dentro de la aplicación, el cliente elige la opción para buscar artículos en el sistema
2. El sistema muestra un formulario para introducir las palabras de búsqueda
3. El cliente introduce los valores
4. El sistema busca los artículos que cumplen con el criterio indicado [Curso alt A]
5. El sistema muestra los artículos en pantalla

Curso alternativo A:
El sistema no encuentra alguna coincidencia
A.4 El sistema muestra en pantalla un mensaje diciendo que ningún elemento coincide
A.5 El caso de uso termina

CU3

Consultar catálogo

Identificador: CU3

Descripción: El cliente navega por la lista de elementos del catálogo disponible

Actores: Usuario y sistema

Precondiciones: Estar dentro de la aplicación de tienda online

Post-condiciones: El usuario navega por la jerarquía de catálogo y su lista de artículos

Disparador: Navegación por la lista de elementos que forman el catálogo

Curso básico de acción:
1. Dentro de la aplicación, el cliente elige la opción para ver el catálogo
2. El cliente pulsa en la opción de productos que le interesa
3. El sistema muestra el contenido de la sección seleccionada por el cliente

CU4

Añadir artículos a solicitud

Identificador: CU4

Descripción: El usuario agrega a la solicitud los artículos que le interesan

Actores: Usuario y sistema

Precondiciones: Haber realizado una búsqueda de artículos en el catálogo, bien por navegación o bien por búsqueda directa y estar registrado en el sistema

Post-condiciones: El artículo seleccionado se añade a una nueva solicitud

Disparador: El cliente selecciona la opción de añadir un artículo a una solicitud

Curso básico de acción:
1. El cliente selecciona el/los artículos en que está interesado
2. El cliente pulsa en añadir el artículo a una solicitud [Curso alt A]
3. El sistema crea una nueva línea en la solicitud
4. El sistema indica al usuario que se ha procedido a añadir el artículo a su solicitud de compra

Curso alternativo A:
El artículo ya existía en la solicitud
A.2 El sistema suma las cantidades anteriores a las actualmente añadidas y no se crea una nueva línea en la solicitud
A.3 El caso de uso continúa en el paso 4 en el curso básico de acción

CU5

Enviar pedido

Identificador: CU5

Descripción: El cliente envía su pedido de artículos deseados

Actores: Usuario y sistema

Precondiciones: El cliente debe haberse identificado en el sistema y deben existir artículos en su solicitud

Post-condiciones: El pedido es creado en el sistema

Disparador: El cliente selecciona la opción de enviar pedido

Curso básico de acción:
1. El cliente selecciona enviar la solicitud actual
2. El sistema crea un pedido a partir de los datos de la solicitud
3. El sistema indica al cliente que se ha procedido a crear el pedido correctamente
